Finance REST API

A simple microservices-based application which exposes a RESTful API, written in Java using the Spring Boot framework. It is intended for use as part of the SESC module at Leeds Beckett University.

version 1.0.0

GET /accounts/student/{studentId}

Used to get an account by the external ID of the student associated with it. See the request sample below. 
                    					
GET /accounts/student/c6666666 HTTP/1.1
Content-Type: application/json;charset=UTF-8
Host: localhost:8080
The following parameter must be provided:
Parameter Description

studentId

The external ID of the student associated with this account.
The API sends back a response similar to the sample below. 
                        				
HTTP/1.1 200 OK
Content-Type: application/hal+json
Content-Length: 475
{
   "id":5,
   "studentId":"c6666666",
   "hasOutstandingBalance":false,
   "_links":{
      "self":{
         "href":"http://localhost:8080/accounts/student/c6666666"
      },
      "accounts":{
         "href":"http://localhost:8080/accounts"
      }
   }
}

The following fields are included in each account returned in the response:

Path Type Description

id

Number

The internal ID of the account.

studentId

String

The external student ID.

hasOutstandingBalance

Boolean

Whether the account has an outstanding balance.

The following links are included in the response:

Relation Description

self

Link to this resource.

accounts

Link to all accounts.

POST /accounts

Used to create a new account for a student. Send a POST request to the /accounts endpoint with a JSON body containing the external student ID. See the request sample below. 
                    					
POST /accounts HTTP/1.1
Content-Type: application/json;charset=UTF-8
Content-Length: 25
Host: localhost:8080

{"studentId": "c3429928"}

GET /invoices/reference/{reference}

Used to get an invoice by its unique external reference. See the request sample below. 
                    					
GET /invoices/reference/BAX5SPP5 HTTP/1.1
Content-Type: application/json;charset=UTF-8
Host: localhost:8080
The following parameter must be provided:
Parameter Description

reference

The external reference of the invoice to return.
The API sends back a response similar to the sample below. 
                        				
HTTP/1.1 200 OK
Content-Type: application/hal+json
Content-Length: 397
{
   "id":10,
   "reference":"BAX5SPP5",
   "amount":10.5,
   "dueDate":"2021-12-25",
   "type":"TUITION_FEES",
   "status":"OUTSTANDING",
   "studentId":"c6666666",
   "_links":{
      "self":{
         "href":"http://localhost:8080/invoices/reference/BAX5SPP5"
      },
      "invoices":{
         "href":"http://localhost:8080/invoices"
      },
      "cancel":{
         "href":"http://localhost:8080/invoices/BAX5SPP5/cancel"
      },
      "pay":{
         "href":"http://localhost:8080/invoices/BAX5SPP5/pay"
      }
   }
}

The following fields are included in each account returned in the response:

Path Type Description

id

Number

The internal ID of the invoice.

reference

String

The external invoice reference used to pay an invoice via the Payment Portal.

amount

Number

The invoice amount.

dueDate

String

The date the invoice must be paid by.

type

String

The type of invoice. Types currently enabled are: LIBRARY_FINE and TUITION_FEES.

status

String

The status of the invoice: OUTSTANDING, PAID or CANCELLED.

studentId

String

The external ID of the student responsible for paying this invoice.

The following links are included in the response:

Relation Description

self

Link to this resource.

invoices

Link to all invoices.

cancel

Link to cancel this invoice.

pay

Link to pay this invoice.

POST /invoices

Used to create a new invoice. Send a POST request to the /invoices endpoint with a JSON body containing the required fields. See the request sample below. 
                    					
POST /invoices HTTP/1.1
Content-Type: application/json;charset=UTF-8
Content-Length: 102
Host: localhost:8080
{
   "amount":15.00,
   "dueDate":"2021-11-06",
   "type":"LIBRARY_FINE",
   "account":{
      "studentId":"c6666666"
   }
}
The following fields must be provided when creating a new invoice:
Path Type Description

amount

Number

The invoice amount.

dueDate

String

The date the invoice must be paid by (yyyy-MM-dd).

type

String

The type of invoice. Types currently enabled are: LIBRARY_FINE and TUITION_FEES.

account.studentId

String

The external ID of the student responsible for paying this invoice.

HTTP/1.1 201 Created

DELETE /invoices/{reference}/cancel

Used to cancel an invoice. Send a DELETE request to the /invoices/{reference}/cancel endpoint. The body of the request must be empty. Only invoices which are currently outstanding can be deleted. See the request sample below. 
                    					
DELETE /invoices/CPDXVWBK/cancel HTTP/1.1
Content-Type: application/json;charset=UTF-8
Host: localhost:8080
The following parameter must be provided when deleting an invoice:
Parameter Description

reference

The external reference of the invoice to delete.
